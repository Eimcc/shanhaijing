# 山海经·异兽录 — 维护方案

## 一、项目结构

```
yaoxiangxiang/
├── index.html          # 页面结构（HTML）
├── style.css           # 所有样式（CSS）
├── app.js              # 精怪数据 + 交互逻辑（JS）
├── images/             # 精怪原画图片
│   ├── 01_jiwei.jpg
│   ├── 02_zheng.jpg
│   ├── ...
│   └── 166_haijun.jpg
└── 维护方案.md          # 本文档
```

## 二、文件职责

| 文件 | 职责 | 修改场景 |
|------|------|----------|
| `index.html` | 页面骨架、布局结构 | 调整页面结构、添加新区域 |
| `style.css` | 颜色、字体、布局、动画 | 调整样式、改配色、响应式适配 |
| `app.js` | 精怪数据、筛选、地图、弹窗 | 增删精怪、修改交互逻辑 |
| `images/` | 精怪Q版水墨原画 | 新增精怪时添加图片 |

## 三、常见维护操作

### 3.1 新增一只精怪

需要修改 **3 个地方**：

#### 第一步：准备图片
- 将图片放入 `images/` 目录
- 命名格式：`{id}_{拼音}.jpg`（如 `167_baihu.jpg`）
- 建议尺寸：1024×1024 像素

#### 第二步：添加数据（app.js）
在 `app.js` 的 `creatures` 数组末尾（`];` 之前）添加：

```javascript
{
    id: 167,                          // 唯一ID，递增
    name: "白虎",                      // 精怪名称
    alias: "西方神兽 · 百兽之王",       // 别名描述
    type: "神兽",                      // 类型：神兽/山精/水怪/凶兽/神鸟/林妖/蛇妖/狐妖
    danger: "★★★★★",                  // 危险等级（1-5星）
    location: "昆仑之山",               // 古籍中的地点
    modernLocation: "新疆昆仑山",       // 对应现代省份
    image: "images/167_baihu.jpg",     // 图片路径
    brief: "有兽焉，其状如虎而白尾。",  // 一句话简介
    appearance: "体型如虎，通体雪白...",  // 外貌描述（2-3句）
    habits: "居于昆仑山巅...",          // 习性描述（2-3句）
    personality: "威严而正义...",        // 性格描述（2-3句）
    quote: "白虎主西方，主秋，主杀。",   // 相关古文引用
    source: "《山海经·西山经》"         // 出处
},
```

**注意事项：**
- 不要使用中文引号 `""`，请用 `「」` 代替
- `id` 必须唯一且递增
- `type` 必须与筛选标签一致（见下方类型列表）

#### 第三步：更新省份映射（app.js）
在 `provinceMap` 对象中找到对应省份，将新 ID 添加到数组中：

```javascript
'新疆': [4, 7, 39, ..., 167],  // 在末尾添加新ID
```

如果省份不存在，需要新建条目，并在地图 SVG 中添加对应省份路径。

### 3.2 删除一只精怪

1. 在 `app.js` 的 `creatures` 数组中找到对应条目，删除整个 `{ ... },` 块
2. 在 `provinceMap` 中找到该精怪的 ID 并从数组中移除
3. 删除 `images/` 目录中对应的图片文件

### 3.3 修改样式

- **改颜色**：编辑 `style.css` 顶部的 `:root` 变量
- **改卡片大小**：修改 `.card-grid` 的 `grid-template-columns` 和 `.card-image-wrapper` 的 `aspect-ratio`
- **改动画**：修改 `@keyframes` 部分或各元素的 `transition` 属性
- **改响应式**：修改 `@media` 查询部分的断点和样式

### 3.4 修改地图

地图 SVG 数据来自阿里云 DataV GeoJSON，如需更新：

```bash
# 重新生成地图SVG路径
python3 /data/user/work/geojson_to_svg.py
# 然后手动替换 index.html 中的 SVG 内容
```

## 五、省份覆盖情况

当前覆盖 29 个省级行政区，地图 SVG 中有 34 个（含北京、天津、上海、香港、澳门等无精怪省份，显示为灰色背景）。

如需新增省份：
1. 在 `provinceMap` 中添加新条目
2. 在地图 SVG 中将对应省份的 class 从 `map-province-bg` 改为 `map-province`
3. 添加 `data-province` 和 `id` 属性

## 七、注意事项

1. **图片命名**：必须以数字ID开头，用下划线分隔，如 `167_baihu.jpg`
2. **中文引号**：JS 中禁止使用 `""` 和 `''`，统一用 `「」`
3. **两处渲染模板**：`app.js` 中有两个卡片渲染函数（`renderCards` 和 `renderCardsByProvince`），修改卡片模板时两处都要同步更新
4. **地图省份名称**：SVG 中的 `data-province` 值必须与 `provinceMap` 的 key 完全一致
5. **文件编码**：所有文件使用 UTF-8 编码
